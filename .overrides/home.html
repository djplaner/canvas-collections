How does Canvas Collections improve Canvas course site design?

Canvas Modules are the Canvas LMS' main tool for organising learning activities. There are long known limitations with Canvas Modules. Canvas Collections introduces three new capabilities to address these.

Modules limitations

collections

The video shows a standard Canvas Modules area to illustrate the known structural, visual, and contextual limitations.

Structure

No way to structure module content (2015) based on purpose. Instead limited to a linear design (2016). The Modules area "is a design nightmare, presenting information as one big list/outline" (2021).

Visual

The Modules area is like "a chain of folders circa Windows 95" (2020). More visual interfaces like tiles or cards (2022) have been requested and extensions developed (2017).

Having more control over the visual representation would help students' visualise priorities (2020), improve visual cues (2020), and enable the modules page to act as a course home page (2016).

Context

Beyond naming modules and their items there is no way to provide additional context about a module. Meaning there is not way to provide connective text (2022) or other module specific information to help students answer questions like why, how, and when should I engage with the module.

Provide more structure

Use Canvas Collections' collections capability to add more structure to the Modules area.

Organise modules into different collections as best fits your design intent. Visitors navigate between different collections. Each collection showing only its modules.

The video shows the same Canvas course's Modules area, but with four collections: Why?, What?, How?, and Questions & Suggestions.

Collections versus collections

Collections (uppercase C) refers to the software package Canvas Collections. collections (lowercase c) refers to the Collections' capability to group modules together.

Provide more visual options

For each collection, select from one of three different representations to move beyond the Windows'95 chain of folders limitation

A collection's representation can be changed to another at any time. More representations can be developed and added.

The video shows the same Canvas course with the same four collections. It starts with each collection having the Collections Only representation. The Collections' configuration is then modified with one collection using the Griffith Cards representation and another the Assessment Table representation.

Learn more about Representations

Provide more context

For each module, add more data about the module.

For example, add a description, a banner, a label, a start date (and perhaps a stop date), and other meta-data. Meta-data that can modify the visual appearance of the modules page through the chosen representation. Use this data to transform vanilla modules into course specific objects (e.g. lecture, workshop, topic etc).

The video shows the same Canvas course now all but that last collection using the Griffith Cards representation. Each module's representation now showing the additional contextual data added to each module, including descriptions, banner images, dates, and labels and label numbers.

Why is this important?

Findability improves self-efficacy & motivation

Findability has been found to be the only and most significant predictor of student motivation and self-efficacy (Simunich et al, 2015; Crews et al, 2017).

Simunich, B., Robins, D. B., & Kelly, V. (2015). The Impact of Findability on Student Motivation, Self-Efficacy, and Perceptions of Online Course Quality. American Journal of Distance Education, 29(3), 174–185. https://doi.org/10.1080/08923647.2015.1058604

Aesthetics & usability improve outcomes & motivation

Heidig et al (2015) found that perceived aesthetics and usability positively affected learner emotional states. Contributing to small improvements in learning outcomes and larger impacts in intrinsic motivation.

Heidig, S., Müller, J., & Reichelt, M. (2015). Emotional design in multimedia learning: Differentiation on relevant design features and their effects on emotions and learning. Computers in Human Behavior, 44, 81–95. https://doi.org/10.1016/j.chb.2014.11.009

Course organisation improves retention

Mulijana & Luo's (2019) systematic literature review found that course organisation improves student satisfaction and motivation, and promotes cognitive presence.

Muljana, P., & Luo, T. (2019). Factors Contributing to Student Retention in Online Learning and Recommended Strategies for Improvement: A Systematic Literature Review. Journal of Information Technology Education: Research, 18. https://doi.org/10.28945/4182

Flexibility necessary for scale

Achieving scale in education requires designs that can flexibly adapt to effective use in a wide variety of contexts across a spectrum of learners and teachers.

Clarke, J., & Dede, C. (2009). Design for Scalability: A Case Study of the River City Curriculum. Journal of Science Education and Technology, 18(4), 353–365. https://doi.org/10.1007/s10956-009-9156-4

How to use Collections?

That depends on your context and your role in your Canvas course site. There are three main steps.

Install (you or your institution)

Configure (you in a teacher or designer role)

Navigate (you as in any role)

There are two ways to install Canvas Collections: individual and institutional. Each intended for a different context. Each with its own advantages and disadvantages. Select the one that best suits your need.

Individual Institutional
Purpose

For teachers/designers to experiment with Collections.

Possibly also to create visual home/landing pages for use without Collections.

For institutions to make Canvas Collections available to all teachers/designers.
Who installs it?

You.

Anyone (with teacher/designer roles in Canvas) is able use the individual installation method.

Your institution.

Typically the institution's Canvas administrator. Since at an institutional level, Collections is installed directly into Canvas and is available to all users.
Who can use it?

The person using the individual installation method can use Collections to configure and navigate courses. Also use it to create visual landing pages.

Once installed institutionally, exactly how people can use Collections will depend on the role they have in Canvas courses.

Teachers/designers will be able to configure Collections for courses. Students will only be able to use Collections, if the course already has Collections configured.

How is it installed?

You install the Canvas Collections userscript into your browser.

An institution's Canvas administrator installs it into Canvas using the Theme Editor
How to do it Install "individual" Collections Install "institutional" Collections

Collections is designed to help improve the information architecture of your course site's Modules. Raising questions like

  1. What is the conceptual model/design intent/information architecture of your course/modules?
  2. How to best use Collections to make clear and actionable to students?

Configuring Collections requires that you first answer those questions and then use the Collections interface to configure Collections to serve your answers.

Learn more about configuring Collections

Once Collections is installed and configured Collections will modify the interface of your course site. There are two types of interface modifications: live Collections and static (aka Claytons) Collections.

Live Static (Claytons)

Collections code is livefor the visitor. It is modifying the Canvas user interface dynamically as the visitor navigates the modules page.

Requires your institutional to have installed Collections in Canvas, or the visitor to have installed Collections as a userscript.

Collections code is not live for the visitor. Instead the course teacher/designer has used Collections to modify one or more Canvas pages with a static version of the Collections' interface.
